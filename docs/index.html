Docker to Home Assistant (D2HA)

D2HA is a modern web UI to monitor and control your Docker containers, with MQTT autodiscovery integration tailored for Home Assistant. It keeps your stack visible, debuggable and easy to manage directly from your browser.

Features
What D2HA adds on top of Docker + Home Assistant
  • Live dashboard for container status, CPU, RAM and network usage ‚Äì see at a glance what your host is doing.
  • One-click controls to start, stop, restart or update containers directly from the web UI.
  • MQTT autodiscovery that publishes sensors and switches following Home Assistant conventions.
  • Safe mode to avoid destructive actions without an explicit confirmation.
  • Performance mode to tune how often the UI refreshes and how heavy the backend polling is.
  • Debug helpers (log views & commands) to understand what Docker is actually doing when you click a button.
How it fits your stack Install D2HA as a Docker container on the same host where you run the rest of your services, point it to your Docker socket and MQTT broker, and it will publish entities that Home Assistant can discover automatically. The goal is to keep your docker-compose world and your configuration.yaml world in sync without manual glue.
Requirements: Docker host (Linux is recommended), MQTT broker compatible with Home Assistant, and a Home Assistant instance to consume the entities.

üöÄ Quick start Up in under 30 seconds

1) Run the container docker run -d -p 12021:12021 -v /var/run/docker.sock:/var/run/docker.sock:ro ghcr.io/arborae/docker2homeassistant:latest Mount the Docker socket in read-only mode. The UI will be available at http://localhost:12021.
2) Secure the dashboard On first login set a new password, optionally enable 2FA and decide if you want Safe Mode to require confirmations for destructive actions.
3) Connect Home Assistant Point D2HA to your MQTT broker to publish sensors and switches automatically. Home Assistant will discover entities without extra YAML.
