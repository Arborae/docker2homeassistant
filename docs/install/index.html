Docker 2 Home Assistant

Install D2HA in minutes

Use this page as a focused install guide for Docker and Docker Compose. Choose the path that best matches your stack and then connect Home Assistant via MQTT if you want autodiscovery.
Docker ready MQTT optional Home Assistant autodiscovery
Recommended setup

üì¶ Docker Compose Recommended

services:
  d2ha:
    image: ghcr.io/arborae/docker2homeassistant:latest
    container_name: d2ha
    restart: unless-stopped
    ports:
      - "12021:12021"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./data:/app/data
    environment:
      D2HA_VERSION: "latest"
      D2HA_SECRET_KEY: "change-this-secret-key"

      # MQTT (optional)
      # MQTT_BROKER: "192.168.1.10"
      # MQTT_PORT: "1883"
      # MQTT_USERNAME: "ha"
      # MQTT_PASSWORD: "password"
      # MQTT_BASE_TOPIC: "d2ha_server"

    # Local build (alternative to image)
    # build:
    #   context: ./d2ha
    #   dockerfile: ./d2ha/Dockerfile
    #   args:
    #     D2HA_VERSION: "latest"
Start with docker compose up -d and update later with docker compose pull followed by docker compose up -d.

üöÄ Quick test (docker run)

docker run -d \
  --name d2ha \
  -p 12021:12021 \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  -v $(pwd)/data:/app/data \
  -e D2HA_SECRET_KEY="change-this-secret-key" \
  ghcr.io/arborae/docker2homeassistant:latest
The ./data folder stores persistent configuration (auth, MQTT preferences, settings). This is ideal for a quick first run.

üåô Nightly builds

image: ghcr.io/arborae/docker2homeassistant:nightly
# or pin a specific commit:
image: ghcr.io/arborae/docker2homeassistant:nightly-<commit_sha>
Nightly images track the latest changes merged into main. Use them if you want early access to new features and are comfortable with faster-moving changes.

üîë First login & onboarding

Username: admin
Password: admin
  • On the first login, a guided wizard will ask you to change the password, configure 2FA and choose Safe mode.
  • From the wizard you can also configure MQTT for Home Assistant autodiscovery.

üåê Access URL

http://localhost:12021
When running on a remote host, replace localhost with the server IP or DNS name (for example http://192.168.1.10:12021).

üì° MQTT .env example

MQTT_BROKER=192.168.1.10
MQTT_PORT=1883
MQTT_USERNAME=homeassistant
MQTT_PASSWORD=password
MQTT_BASE_TOPIC=d2ha_server
MQTT_DISCOVERY_PREFIX=homeassistant
MQTT_NODE_ID=d2ha_server
MQTT_STATE_INTERVAL=5
Without MQTT, D2HA is still a full Docker dashboard. MQTT is only required if you want Home Assistant to discover entities automatically.
Pro tip: define a strong D2HA_SECRET_KEY from day one and enable 2FA during the first login. This secures the web UI that controls your containers.
